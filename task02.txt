Git と GitHub を使った共同開発のワークフローを説明しなさい

その際、それぞれのキーワードを関連付けて、自分の言葉で以下の説明をしてください

### キーワード

---

ローカルリポジトリ、リモートリポジトリ、ベアリポジトリ、クローン、ブランチ、コミット、フェッチ、マージ、コンフリクト、スタッシュ、プル、プッシュ、プルリクエスト



GitHubにベアリポジトリのリモートリポジトリを作り、作業場所に、
リモートリポジトリのデータをクローンします。
マスターブランチから機能開発やバグの修正などのブランチを作成して、
コミットをしながら開発を進めます。
マスターブランチにローカルリポジトリのデータを反映させるためにプッシュを行い、プルリクエストを
出し、問題がなければマージをします。

コンフリクトが起きた場合、他の開発者と相談をし解決をします。

リモートリポジトリのデータをローカルリポジトリに反映させるときは、
フェッチをしてマージをするか、プルをします。

ブランチの移動をする際、現在のブランチでコミットしていないデータが
存在している場合、移動先のブランチで必要なければスタッシュを行い、
コミットしていないデータを保存します。
